### ⬡ Design principles I actually follow

```
01  The domain owns the rules. Infrastructure is a detail.
02  If you can't test it without the framework, the boundary is wrong.
03  Errors should mean something. Generic 500s are a design failure.
04  Add complexity only when the system earns it.
05  Document the decision, not just the code.
```
